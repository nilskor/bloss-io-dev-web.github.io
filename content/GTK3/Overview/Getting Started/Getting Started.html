Getting Started with GTK

GTK is a widget toolkit. Each user interface created by GTK consists of widgets. This is implemented in C using GObject, an object-oriented framework for C.

Summary

Widgets are organized in a hierachy. The window widget is the main container. The user interface is then built by adding buttons, drop-down menus, input fields, and other widgets to the window. If you are creating complex user interfaces it is recommended to use GtkBuilder and its GTK-specific markup description language, instead of assembling the user interface (UI) manually. You can also use a visual user interface editor (for example Glade).

GTK is event-driven. The toolkit listens for events such as clicks on a button, and passes those events on to your application to handle.

This chapter contains some tutorial information to get you started with GTK programming. It assumes that you have installed GTK, its dependencies, and a C compiler ready for use. (If you need to build the GTK library itself first, then refer to the Compiling GTK Applications section in this reference.

Packing Widgets in a Layout

When creating an application, you'll want to put more than one widget inside a window. When you want to put more than one widget into a window, it becomes important to control how each widget is positioned and sized. This is where packing comes in.

Building User Interfaces

When construcing a more complicated user interface, with dozens or hundreds of widgets, doing all the setup work in C code is cumbersome, and making changes becomes more difficult and prone to errors. To assist with this, GTK supports the separation of user interface layout from your business logic, by using UI descriptions in an XML format that can then be parsed by the GtkBuilder class.

Building applications

A typical application consists of numerous files. GTK includes application support that is built on top of GApplication.

Custom Drawing
